Google Protocol Buffer Explained

Google Protocol Buffer (Proto)

You can read what Protocol Buffers (Proto, Protobuf) are at Google’s documentation. NIRA uses them as the return response from the HERE-tile APIs (it is also used for Mapbox Vector tile) to package the data in a compact format. The output from these APIs will be in binary format which you will need a declaration of the .proto format used to decode.

NIRA supplies different .proto files depending on the API used. See list below.

Protocol Buffer Files

  • HERE-tile offset
    • here-tile-offset.zip
  • HERE-tile sublink
    • here-tile-sublink.zip
  • Mapbox vector tile
    • Provided by Mapbox, but also found here vector_tile.proto

Decoding NIRA Protocol Buffer

Get a hold of a binary data file from the NIRA API, see testing the API.

These binary files can then be decoded using code or the program protoc. Unpack the protobuf definition zip-file corresponding to the API the binary data file came from. Move the binary file to the extracted folder and rename it response.bin Standing inside the extracted folder one can then call the following commands:

Here Tile Offset

If response.bin is an roadlayer tile
protoc \
    -I Path/to/here-tile-offset/ --decode se.niradynamics.ncs.protobuf.output.RoadLayerTile \
    se/niradynamics/ncs/protobuf/output/RoadLayerTile.proto \
    < response.bin
If response.bin is a roadlayer-geometry tile
protoc \
    -I Path/to/here-tile-offset/ --decode se.niradynamics.ncs.protobuf.output.RoadLayerGeometryTile \
    se/niradynamics/ncs/protobuf/output/RoadLayerGeometryTile.proto \
    < response.bin

Here Tile Sublink

If response.bin is an aggregated tile
protoc \
    -I Path/to/here-tile-sublink/ --decode se.niradynamics.ncs.protobuf.aggregated.Tile \
    se/niradynamics/ncs/protobuf/aggregated/AggregatedTile.proto \
    < response.bin
If response.bin is a geometry tile
protoc \
    -I Path/to/here-tile-sublink/ --decode se.niradynamics.ncs.protobuf.geometry.Tile \
    se/niradynamics/ncs/protobuf/geometry/GeometryTile.proto \
    < response.bin

Mapbox Vector Tile

protoc \
    -I Path/to/directory/ --decode vector_tile.Tile \
    vector_tile.proto \
    < response.bin
